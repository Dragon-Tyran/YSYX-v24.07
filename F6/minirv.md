### 其他四条指令

> ![](attachments/Pasted%20image%2020260701133622.png)





#### 1. `lw`（Load Word）—— 从内存读 4 字节到寄存器
- **Load（读内存 → 寄存器）**：I-type 编码
```
lw   rd, offset(rs1)     # rd = Mem[rs1 + sign_extend(imm)]，读4字节（word），有符号
```

**举例**
```asm
lw a0, 0(a1)     # 假设 a1 = 100
```

**含义**：从 `a1 + 0 = 100` 这个地址开始，**连续读4个字节**，凑成一个完整的32位数，放进 `a0`。也就是从RAM中读一个字


```
从地址100开始读4字节：[12][34][56][78]
拼成32位数放入a0：   a0 = 0x78563412  (小端序拼法)
```

#### 2. `sw`（Store Word）—— 把寄存器的4字节写到内存

- **Store（寄存器 → 写内存）**：S-type 编码

```
sw   rs2, offset(rs1)    # Mem[rs1 + imm] = rs2，写4字节
```
**举例**

```asm
sw a0, 0(a1)     # 假设 a1 = 200, a0 = 0x78563412
```

**含义**：把 `a0` 寄存器里的完整32位值，**拆成4字节写入**从地址200开始的4个内存格子。

```
写入前：地址200-203 原来是 [00][00][00][00]
写入后：地址200-203 变成   [12][34][56][78]
```





#### 3. `lbu`（Load Byte Unsigned）—— 只从内存读 1 字节，放进寄存器（高位补0）
- **Load（读内存 → 寄存器）**：I-type 编码
```  
lbu  rd, offset(rs1)     # rd = Mem[rs1 + sign_extend(imm)]，读1字节（byte），无符号扩展
```
**举例**

```asm
lbu a0, 0(a1)    # 假设 a1 = 100，地址100处存的是 0x12
```

**含义**：只读**1个字节**（而不是4个），然后把这1字节放进32位寄存器的最低8位，**寄存器剩下的高24位全部补0**。

```
地址100: [12]
读进a0后：a0 = 0x00000012   （高24位补0）
```

为什么要补0？因为寄存器是32位的，而你只读了8位数据，剩下24位必须有个说法——`lbu` 的规则是"当作无符号数"，所以直接补0。

**用途**：读单个字符（比如ASCII字符）、字节数组、字符串处理的时候用得非常多。

---

#### 4. `sb`（Store Byte）—— 只把寄存器的最低1字节写到内存
- **Store（寄存器 → 写内存）**：S-type 编码

```
sb   rs2, offset(rs1)    # Mem[rs1 + imm] = rs2的低8位，写1字节
```

**举例**

```asm
sb a0, 0(a1)     # 假设 a0 = 0x000000AB, a1 = 300
```

**含义**：只取 `a0` 寄存器**最低的8位**，写到内存地址300这一个格子，寄存器高位的内容完全不管、不写出去。

```
a0 = 0x000000AB
写入内存地址300：[AB]        （只写最低1字节，高位0000AB的高24位被丢弃不写）
```





#### 逐条说明

|指令|全称|功能|
|---|---|---|
|**lw**|Load Word|从内存读 **4字节（32位）**，直接存入寄存器|
|**lbu**|Load Byte Unsigned|从内存读 **1字节（8位）**，**无符号扩展**到32位（高24位补0）|
|**sw**|Store Word|把寄存器里的 **4字节** 写到内存|
|**sb**|Store Byte|只把寄存器里**最低1字节**写到内存（高位丢弃不写）|
